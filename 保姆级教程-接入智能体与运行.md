# 保姆级教程：运行前端 + 接入智能体

面向完全零基础：从安装软件到看到真实接口回复，按顺序做即可。

---

## 一、你需要先装什么

1. **Node.js**（自带 `npm`）  
   - 打开 [https://nodejs.org](https://nodejs.org) ，下载 **LTS** 版本并安装。  
   - 安装完成后打开终端（Mac：「终端」；Windows：「命令提示符」或 PowerShell），输入：

```bash
node -v
npm -v
```

两行都能显示版本号（如 `v20.x`）就说明 OK。

2. **（可选）Python 3**  
   - 仅在使用本仓库自带的 **模拟智能体脚本** 时需要。  
   - 终端输入 `python3 --version` 有版本号即可。

---

## 二、第一次运行前端（不接智能体也能玩）

1. 打开终端，进入项目目录（按你电脑上的实际路径改）：

```bash
cd /Users/catthunder/career-agent-hub
```

2. 安装依赖（只需做一次，以后要再运行可跳过）：

```bash
npm install
```

3. 启动开发服务器：

```bash
npm run dev
```

4. 终端里会出现类似：

```text
  Local:   http://localhost:5173/
```

用浏览器打开这个地址。  
此时 **没有配置智能体** 时，右侧聊天会显示 **「演示模式」** 占位文字，这是正常的。

5. 停止服务：在运行 `npm run dev` 的终端里按 `Ctrl + C`。

---

## 三、智能体接口约定（你的后端要长什么样）

前端会向一个地址发 **POST** 请求，**JSON 请求体**结构如下（与 `src/lib/agentClient.ts` 一致）：

```json
{
  "messages": [
    { "role": "system", "content": "可选系统提示" },
    { "role": "user", "content": "用户说的话" },
    { "role": "assistant", "content": "助手上一句" }
  ],
  "context": {
    "resume_excerpt": "简历正文前 8000 字以内",
    "mode": "修改简历 | 模拟面试 | 职业匹配 等当前 Tab 名称"
  }
}
```

**响应**支持两种形式（二选一）：

1. **JSON**，且至少包含下面**任意一个**字段（前端按顺序取第一个非空的）：  
   - `reply`  
   - `message`  
   - `content`  

2. **纯文本**（`Content-Type` 不是 `application/json`），整段 body 当作助手回复。

若 HTTP 状态码不是 2xx，前端会把错误信息显示在聊天里。

## 四、接入方式 A：直连你的线上/局域网接口（需处理跨域）

适合：你的智能体 API 已经部署在公网或内网，且 **允许浏览器跨域（CORS）**。

1. 在项目根目录（与 `package.json` 同级）新建文件 **`.env`**（注意开头有点，没有 `.txt` 后缀）。

2. 写入一行（把地址换成你自己的）：

```env
VITE_AGENT_URL=https://你的域名/你的路径
```

3. **保存 `.env` 后，必须重新启动** `npm run dev`（环境变量只在启动时读取）。

4. 若浏览器控制台报 **CORS** 错误，需要在你的智能体服务端加上响应头，例如：

- `Access-Control-Allow-Origin: *`（或指定你的前端域名）  
- `Access-Control-Allow-Headers: Content-Type`  
- `Access-Control-Allow-Methods: POST, OPTIONS`  

并对 **OPTIONS** 预检请求返回 204。

---

## 五、接入方式 B：开发时用 Vite 代理（推荐本地调试，避开 CORS）

浏览器只访问 `http://localhost:5173`，请求发到同源路径 **`/api/agent`**，由 Vite **转发**到你本机的智能体服务，一般 **不需要** 给智能体配 CORS。

1. 在 **`.env`** 里写：

```env
VITE_AGENT_URL=/api/agent
VITE_AGENT_PROXY_TARGET=http://127.0.0.1:8000
VITE_AGENT_PROXY_PATH=/chat
```

含义简要说明：

| 变量 | 作用 |
|------|------|
| `VITE_AGENT_URL` | 前端 `fetch` 用的路径，固定写 `/api/agent` |
| `VITE_AGENT_PROXY_TARGET` | 你的后端根地址，如本机 `8000` 端口 |
| `VITE_AGENT_PROXY_PATH` | 转发后的路径，例如后端是 `POST /chat` 就写 `/chat` |

若你的接口是 **`POST http://127.0.0.1:8000/`**（根路径），把最后一行改成：

```env
VITE_AGENT_PROXY_PATH=/
```

2. 保存后 **重新** `npm run dev`。

3. 实际效果：浏览器请求  
   `POST http://localhost:5173/api/agent`  
   → 被代理为  
   `POST http://127.0.0.1:8000/chat`（与上面 PATH 一致）。

## 六、用仓库自带脚本「假智能体」跑通全流程（推荐第一步）

1. 开**第一个**终端，启动模拟服务：

```bash
cd /Users/catthunder/career-agent-hub
python3 scripts/mock_agent_server.py
```

看到 `Mock agent: http://127.0.0.1:8000/chat` 即成功。

2. 在项目根目录创建 **`.env`**（内容同 **五、接入方式 B** 的三行）。

3. 开**第二个**终端：

```bash
cd /Users/catthunder/career-agent-hub
npm run dev
```

4. 浏览器打开 `http://localhost:5173`，在右侧智能体里发一句话，应收到 **「【模拟智能体】已收到请求」** 的 JSON 回复，而不是「演示模式」。

5. 把你的真实大模型调用写进后端，替换 `mock_agent_server.py` 的逻辑即可；**保持同样的请求/响应格式**最省事。

## 七、打包上线（可选）

```bash
npm run build
```

生成目录 **`dist/`**，把 `dist` 里的静态文件放到 Nginx、OSS、Vercel 等即可。

注意：**线上环境**同样要在构建时或运行平台配置 **`VITE_AGENT_URL`**。若前端和 API **不同域**，仍需 API 端配置 **CORS**，或让网关同源反代。

## 八、常见问题

**Q：改完 `.env` 没生效？**  
A：一定要 **关掉再重新执行** `npm run dev`。

**Q：一直显示「演示模式」？**  
A：`VITE_AGENT_URL` 为空或未以 `VITE_` 开头；或变量名写错。检查 `.env` 是否在项目根目录。

**Q：提示网络错误 / Failed to fetch？**  
A：后端没启动、端口不对、代理 `TARGET`/`PATH` 写错，或 HTTPS 证书问题。先看浏览器 **开发者工具 → Network** 里失败的请求 URL。

**Q：我的接口字段名不一样？**  
A：改 `src/lib/agentClient.ts` 里 `fetch` 的 `body` 和解析 JSON 的字段名，或在你的网关里做一层转换。

## 九、相关文件速查

| 文件 | 作用 |
|------|------|
| `src/lib/agentClient.ts` | 请求体格式、解析回复 |
| `vite.config.ts` | 开发代理 `/api/agent` |
| `.env` | 本地配置（勿提交密钥到公开仓库） |
| `scripts/mock_agent_server.py` | 本地假后端 |

按 **二 → 六** 顺序做一遍，即可在不接真实大模型的情况下跑通；再把模拟脚本换成你的智能体即可。
